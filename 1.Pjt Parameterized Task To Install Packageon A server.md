This task wants you to automate package installation on your environment's Storage Server (ststor01) using Jenkins.

Instead of manually logging into the storage server and typing commands, Jenkins will act as the orchestrator. You will build a parameterized job that allows anyone to type a package name (like vim-enhanced), click "Build", and Jenkins will log into the storage server over SSH to install it automatically.

Phase 1: Check the Storage Server Credentials
Before doing anything in Jenkins, click the Details of all Users and Servers button at the top of your lab screen to find the exact username, password, and hostname for the Storage Server.

Usually, the credentials are: Hostname: ststor01.stratos.xfusioncorp.com | Username: natasha | Password: Bl@kW.

Phase 2: Configure Jenkins (SSH Setup)
Jenkins needs a plugin and remote server configuration to communicate with the storage server.

Click the Jenkins button in the top menu bar of your lab interface.

Log in with Username: admin and Password: Adm!n321.

Install the SSH Plugin:

Go to Manage Jenkins > Plugins (or Manage Plugins) > Available plugins.

Search for SSH (look for the standard SSH plugin that lets you execute commands on a remote host).

Check the box, click Install, and check Restart Jenkins when installation is complete.

Wait a minute, refresh the page, and log back in.

Add the Server Credentials:

Go to Manage Jenkins > Credentials > System > Global credentials (unrestricted).

Click Add Credentials.

Kind: Username with password

Username: natasha (or the username from the details tab)

Password: Bl@kW (or the password from the details tab)

ID: storage-server (or anything memorable)

Click Create.

Add the SSH Remote Host:

Go to Manage Jenkins > System (or Configure System).

Scroll down to the SSH Remote Hosts section and click Add.

Hostname: ststor01.stratos.xfusioncorp.com

Port: 22

Credentials: Select the natasha credential you just created.

Click Check Connection to ensure it successfully logs in, then click Save.

Phase 3: Create the Parameterized Job
Now we will create the actual job that takes the package name as a variable parameter ($PACKAGE).

On the Jenkins dashboard, click New Item.

Name it exactly: install-packages

Select Freestyle project and click OK.

Under the General section, check the box: This project is parameterized.

Click Add Parameter and select String Parameter.

Name: PACKAGE

Scroll down to the Build Steps section (or Build).

Click Add build step and choose Execute shell script on remote host using SSH.

SSH Site: Select your storage server (natasha@ststor01...) from the dropdown.

In the Command text area, write the command to pass your password dynamically to sudo along with the variable:

Bash
echo 'Bl@kW' | sudo -S yum install -y $PACKAGE
(Note: replace Bl@kW with the actual password found in your user details tab if it differs).

Click Save.

Phase 4: Execute the Job
On the job's main page, click Build with Parameters on the left menu.

Inside the PACKAGE field, enter the default requested package name: vim-enhanced

Click Build.

Once the build finishes appearing in the history on the left side, click the build number, select Console Output, and verify it successfully logs into the storage server and finishes the installation without errors.

Once verified, click the Check validation button on the lab interface!
