### This task wants you to automate package installation on your environment's Storage Server  using Jenkins.
Instead of manually logging into the storage server and typing commands, Jenkins will act as the orchestrator. You will build a parameterized job that allows anyone to type a package name (like vim-enhanced), click "Build", and Jenkins will log into the storage server over SSH to install it automatically.

#### Phase 1: Check the Storage Server Credentials
Before doing anything in Jenkins, click the Details of all Users and Servers button at the top of your lab screen to find the exact username, password, and hostname for the Storage Server.



#### Phase 2: Configure Jenkins (SSH Setup)
- 1.Jenkins needs a plugin and remote server configuration to communicate with the storage server.
Log in Jenkins  
- 2.Install the SSH Plugin:  
- 3. Add the Server Credentials:
     Go to Manage Jenkins > Credentials > System > Global credentials (unrestricted).
    Click Add Credentials.
```
       Kind: Username with password
       Username:user
       Password: pass
       ID: nameOfCred
```
- 4. Add the SSH Remote Host:
     Go to Manage Jenkins > System (or Configure System). --> SSH Remote hOST (Scroll to Bottom)

```    
    Hostname: storage01
    Port: 22
    Credentials: Select the credential you just created.
```


#### Phase 3: Create the Parameterized Job
Now we will create the actual job that takes the package name as a variable parameter ($PACKAGE).

- 1. On the Jenkins dashboard, click New Item.
      * Name it exactly: install-packages
      * Select Freestyle project and click OK.
      * Under the General section, check the box: This project is parameterized.
        **Click Add Parameter and select String Parameter.**
          - Name: PACKAGE

- 2.Scroll down to the Build Steps section (or Build).
   Click Add build step and choose Execute shell script on remote host using SSH.

     SSH Site: Select your storage server  from the dropdown.

In the Command text area, write the command to pass your password dynamically to sudo along with the variable:

```Bash
echo 'password' | sudo -S yum install -y $PACKAGE
(Note: Here we fed directly password , A safer alternative is to configure sudoers rules for the specific command, or use Jenkins credentials management to inject secrets securely.).
```
Click Save.

#### Phase 4: Execute the Job
On the job's main page, click Build with Parameters on the left menu.
- Inside the PACKAGE field, enter the default requested package name: vim-enhanced
- Click Build.
- Once the build finishes appearing in the history on the left side, click the build number, select Console Output, and verify it successfully logs into the storage server and finishes the installation without errors.
